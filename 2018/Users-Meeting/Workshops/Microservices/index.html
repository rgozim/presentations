Image Data Resource (IDR)

  • Publically available

  • Highly flexible

  • Big data

What is the "Data" in IDR

  • Images

  • Metadata

The problem

  • 46 terabytes (TB) of data
  • 160 gigabytes (GB) database
  • 50 million images

Performance

  • Time taken to complete a some action
  • What has to happen to obtain data

Microservices

  • Small, independently deployable, modular services
  • Scalable
  • Reduced complexity

Disadvantages of decoupled clients

  • Discoverability: Adding feature pages to website
  • More steps to install "everything"
  • Devs: More repos and testing infrastructure to maintain

Testing of decoupled clients

  • Use omero-test-infra
  • Docker-composed OMERO server for testing
  • Testing scripts run by Travis CI on every github PR
